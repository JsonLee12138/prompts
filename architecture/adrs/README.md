> Architecture Decision Records - 记录重要的架构决策

---

## 📚 什么是 ADR？

架构决策记录 (ADR) 是用于记录架构决策的轻量级文档。每个 ADR 包含：

- **决策背景**：为什么需要做这个决策
- **决策内容**：具体的选择和方案
- **决策后果**：正面和负面的影响
- **替代方案**：考虑过的其他选项

---

## 🎯 何时创建 ADR？

创建 ADR 的典型场景：

- 引入新技术或框架
- 改变架构模式（如从单体到微服务）
- 重要的性能优化决策
- 安全相关的决策
- 数据存储方案变更
- 通信协议选择
- 重大重构决策

## 🔖 ADR 状态说明

- **提议 (Proposed)**：正在讨论的决策
- **已接受 (Accepted)**：决策已批准，正在实施
- **已实施 (Implemented)**：决策已完成实施
- **已弃用 (Deprecated)**：决策已过时，被新决策替代
- **已拒绝 (Rejected)**：提议但未被采纳

---

## ✅ ADR 质量检查清单

创建 ADR 时请检查：

- [ ] 标题清晰描述决策
- [ ] 背景说明充分
- [ ] 考虑了至少 2 个替代方案
- [ ] 正面和负面后果都已记录
- [ ] 决策标准明确
- [ ] 相关利益相关者已知晓
- [ ] 文档已存档并可访问
